# 仪表盘（Dashboard）详细设计

> 本文档描述仪表盘模块的交互细节。

---

## 1. 关键指标卡片

### 1.1 指标示例

至少 4 个指标卡：

| 指标 | 说明 |
|------|------|
| 今日总请求数 | 当天累计请求数量 |
| 最近 24 小时成功率 | 请求成功百分比 |
| 平均响应时间 | 请求平均耗时 |
| 活跃会话数 / 在线用户数 | 当前活跃用户 |

### 1.2 卡片内容

每个卡片包含：
- 标题
- 主数值
- 对比信息（与昨天相比的变化趋势）
- 百分比和趋势方向箭头

### 1.3 视觉要求

- 支持玻璃拟态效果
- 半透明背景、模糊背景、悬浮感
- 指标变化用颜色区分：
  - 上涨：绿色
  - 下跌：红色

---

## 2. 趋势图

### 2.1 基本要求

- 至少 1 个随时间变化的图表（折线图或面积图）
- 展示最近一段时间的关键指标趋势
- 例如：最近 7 天的请求量 / 成功量 / 错误量

### 2.2 视觉要求

- 图表配色需要与主题风格协调

---

## 3. 复杂图表（必须）

### 3.1 图表类型

除基础趋势图外，至少再提供 **3 类复杂图表**：

| 图表类型 | 用途 | 实现方式 |
|---------|------|----------|
| 组合图（柱状 + 折线双轴） | 展示请求量与成功率同屏对比 | Recharts ComposedChart |
| 堆叠图（Stacked Bar/Area） | 展示不同渠道或模块的贡献占比 | Recharts StackedArea/Bar |
| 漏斗图/雷达图/热力图 | 业务深度分析 | 任选其一 |

### 3.2 交互要求

- 图表必须具备交互性：
  - tooltip
  - legend
  - 数据高亮
  - 筛选

### 3.3 主题要求

- 支持主题色跟随
- 配色与主题风格协调

---

## 4. 分类统计图

### 4.1 基本要求

- 至少 1 个分类维度的图表（柱状图 / 饼图等）
- 展示不同业务模块、不同错误类型或不同接口的占比

### 4.2 交互要求

- 饼图需要支持交互式 tooltip 和图例

---

## 5. 最近事件列表

### 5.1 字段定义

| 字段 | 说明 |
|------|------|
| 时间 | 事件发生时间 |
| 类型 | 事件类型 |
| 描述 | 事件描述 |
| 影响范围 | 受影响的范围 |
| 状态标签 | 事件状态 |

### 5.2 视觉要求

- 支持玻璃拟态效果
- 不同类型事件用不同颜色标签区分：
  - 错误：红色
  - 成功：绿色
  - 警告：黄色

---

## 6. 时间范围过滤

### 6.1 快捷选项

- 今天
- 最近 7 天
- 最近 30 天
- 自定义范围

### 6.2 交互要求

- 切换时间范围时，指标卡片和图表展示的数据要跟随变化
- 支持日期范围选择器

---

## 7. 插件挂载区域

### 7.1 功能说明

- 预留一个区域，用于动态加载已启用插件的组件内容
- 例如：插件提供的小组件、小卡片等

### 7.2 状态处理

| 插件状态 | 区域显示 |
|---------|----------|
| 插件启用 | 显示插件提供的组件内容 |
| 插件禁用 | 显示占位内容或隐藏 |

### 7.3 布局要求

- 挂载区域需要与整体布局协调
- 不能显得突兀
