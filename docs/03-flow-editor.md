# 流程编排（Flow Editor）详细设计

> 本文档描述流程编排模块的交互细节，基于 @xyflow/react 实现。

---

## 1. 流程列表页面

### 1.1 列表字段

| 字段 | 说明 |
|------|------|
| 流程名称 | 流程的名称 |
| 流程描述 | 流程的描述信息 |
| 状态 | 启用 / 禁用 / 草稿 |
| 创建时间 | 流程创建时间 |
| 最近更新时间 | 最后修改时间 |

### 1.2 操作支持

| 操作 | 说明 |
|------|------|
| 新建流程 | 进入编辑画布 |
| 编辑流程 | 进入编辑画布 |
| 复制流程 | 复制现有流程 |
| 删除流程 | 需确认 |
| 启用/禁用流程 | 切换流程状态 |

### 1.3 其他功能

- 支持搜索和筛选（按状态、名称）
- 列表支持分页

---

## 2. 流程编辑页面（画布）

### 2.1 画布交互

| 操作 | 说明 |
|------|------|
| 鼠标滚轮缩放 | 缩放画布视图 |
| 拖拽移动画布 | 平移画布视图 |
| 触摸板操作 | 支持触摸手势 |

### 2.2 节点操作

| 操作 | 说明 |
|------|------|
| 添加节点 | 在画布中添加新节点 |
| 拖动节点 | 移动节点位置 |
| 连接节点 | 在节点之间创建连线 |

### 2.3 节点添加方式（重要）

**主要方式：拖拽投放**

- 左侧节点面板可拖拽节点模板到画布落点创建节点
- 必须是真实拖拽到画布落点生成节点
- 不能仅用点击按钮替代主流程

**节点面板要求**：

| 要求 | 说明 |
|------|------|
| 分组展示 | 按功能分组（基础节点、逻辑节点、执行节点） |
| 图标 | 每个节点模板都必须带有图标 |
| 紧凑样式 | 避免单项过高导致面板可视数量过少 |

**辅助方式**：保留快捷操作（按钮添加）

### 2.4 画布辅助功能

| 功能 | 说明 |
|------|------|
| 缩略图（MiniMap） | 可快速定位，需要可见、可交互 |
| 网格对齐 | 可开关的对齐功能 |
| 主题协调 | 画布背景需要与主题协调 |

### 2.5 布局要求

- 画布区域应是编辑页的视觉主体
- 宽度和高度都需要明显大于左右辅助面板
- 缩略图需要能帮助用户在大画布中快速回到目标区域

## 3. 节点类型

### 3.1 节点类型定义

至少支持 **6 种节点类型**：

| 节点类型 | 说明 | 约束 |
|---------|------|------|
| 开始节点（Start） | 流程入口 | 只能有一个 |
| 结束节点（End） | 流程出口 | 可以有多个 |
| 任务节点（Task） | 执行具体任务 | - |
| 条件分支节点（Condition） | 根据条件选择分支 | - |
| 并行节点（Parallel） | 同时执行多个分支 | - |
| 循环节点（Loop） | 重复执行直到条件满足 | - |

### 3.2 视觉要求

- 每种节点类型需要有独特的视觉样式
- 包括：颜色、图标、形状
- 节点需要支持主题色

---

## 4. 节点属性编辑

### 4.1 编辑方式

点击节点时，弹出编辑框：
- Modal（模态框）
- Drawer（抽屉）
- 侧边面板

### 4.2 可编辑属性

| 属性 | 说明 |
|------|------|
| 节点名称 | 节点的显示名称 |
| 节点描述 | 节点的描述信息 |
| 节点类型特定属性 | 根据节点类型不同而不同 |

### 4.3 节点类型特定属性

| 节点类型 | 特定属性示例 |
|---------|-------------|
| 任务节点 | 执行方式、超时设置 |
| 条件节点 | 条件表达式 |
| 循环节点 | 循环条件、最大次数 |

### 4.4 交互要求（重要）

| 操作 | 说明 |
|------|------|
| 单击节点 | 选中节点 |
| 双击节点 | 直接打开节点属性编辑框（高频快捷操作） |

> 单击选中与双击编辑需要区分，避免用户必须依赖额外按钮才能进入编辑

---

## 5. 连线编辑

### 5.1 连线属性

| 属性 | 说明 |
|------|------|
| 连线标签 | 连线的显示文本 |
| 条件表达式 | 触发该连线的条件 |
| 连线样式 | 颜色、线型（实线/虚线/点线） |
| 箭头 | 指示方向（必须有） |

### 5.2 条件分支连线

- 条件分支节点的连线需要显示条件标签

### 5.3 交互要求（重要）

| 操作 | 说明 |
|------|------|
| 双击连线 | 直接打开连线编辑框，修改标签、条件和线型 |

---

## 6. 浮动工具栏（Floating Toolbar）

### 6.1 触发条件

- Hover 到节点时显示
- Hover 到连线时显示

> 节点与连线 Hover 时都应能触发对应的浮动工具栏，不能只在选中后显示

### 6.2 显示与隐藏逻辑（重要）

| 行为 | 说明 |
|------|------|
| 显示触发 | 鼠标进入节点/连线区域 |
| 隐藏延迟 | 鼠标离开后延迟 160ms 隐藏 |
| 工具栏 Hover 保持 | 鼠标移动到工具栏上时保持显示 |
| 工具栏离开隐藏 | 鼠标离开工具栏后延迟隐藏 |

> 延迟隐藏和工具栏 hover 保持是为了避免用户移动鼠标到工具栏按钮时工具栏消失

### 6.3 工具栏操作

| 操作 | 说明 |
|------|------|
| 删除 | 删除当前节点/连线 |
| 快速复制 | 复制当前节点 |
| 快速跳转编辑 | 打开属性编辑面板 |

### 6.4 视觉要求

- 工具栏需要跟随主题风格
- 浮动工具栏应尽量贴近当前节点或连线
- 减少视线跳转

---

## 7. 编辑器效率能力（必须）

### 7.1 快捷键

| 功能 | 快捷键 | 说明 |
|------|--------|------|
| Undo | Ctrl/Cmd+Z | 撤销 |
| Redo | Ctrl/Cmd+Y | 重做 |
| Copy | Ctrl/Cmd+C | 复制节点 |
| Paste | Ctrl/Cmd+V | 粘贴节点 |
| Delete | Delete 键 | 删除选中节点/连线 |

### 7.2 历史记录（Undo/Redo）

| 配置 | 值 |
|------|-----|
| 最大历史记录数 | 50 条 |
| 超出策略 | 保留最近的 50 条 |

**历史记录触发条件**：

| 变化类型 | 是否记录 | 说明 |
|---------|---------|------|
| 节点拖拽中 | 否 | 拖拽过程中不记录 |
| 节点拖拽结束 | 是 | dragging=false 时记录 |
| 节点选择变化 | 否 | 选择不影响历史 |
| 节点尺寸变化 | 否 | 尺寸变化不记录 |
| 连线选择变化 | 否 | 选择不影响历史 |
| 连线添加/删除 | 是 | 记录连线变化 |

### 7.3 其他功能

| 功能 | 说明 |
|------|------|
| 导出 JSON | 导出当前流程为 JSON 格式 |
| 恢复保存版本 | 一键恢复到最近保存版本 |

---

## 8. 页面离开保护

### 8.1 未保存变更检测

当页面有以下未保存的变更时：
- 节点位置变化
- 节点属性变化
- 连线变化
- 节点增删

### 8.2 离开保护行为

| 场景 | 行为 |
|------|------|
| 浏览器刷新/关闭 | 弹出浏览器原生确认对话框 |
| 路由跳转 | 需要弹出确认对话框（应用内） |

### 8.3 脏状态标识

- 页面有未保存修改时，需要显示脏状态标识
- 保存后清除脏状态

## 8. 数据结构

### 8.1 流程定义

```typescript
interface FlowDefinition {
  id: string;
  name: string;
  description?: string;
  status: 'enabled' | 'disabled' | 'draft';
  nodes: FlowNode[];
  edges: FlowEdge[];
  createdAt: Date;
  updatedAt: Date;
}
```

### 8.2 节点定义

```typescript
interface FlowNode {
  id: string;
  type: NodeType;
  position: { x: number; y: number };
  data: {
    label: string;
    description?: string;
    config?: Record<string, unknown>;
  };
}

type NodeType = 'start' | 'end' | 'task' | 'condition' | 'parallel' | 'loop';
```

### 8.3 连线定义

```typescript
interface FlowEdge {
  id: string;
  source: string;
  target: string;
  label?: string;
  condition?: string;
  style?: 'solid' | 'dashed' | 'dotted';
}
```
