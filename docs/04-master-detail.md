# 主子表管理（Master-Detail CRUD）详细设计

> 本文档描述主子表管理模块的交互细节，展示复杂的数据编辑场景。

---

## 1. 主表列表页面

### 1.1 列表功能

| 功能 | 说明 |
|------|------|
| 多选 | 复选框列 |
| 排序 | 点击表头排序 |
| 分页 | 页码选择器、每页条数选择 |
| 刷新 | 刷新列表数据 |

### 1.2 交互要求

| 交互 | 说明 |
|------|------|
| 点击行或"查看明细" | 在多标签页导航中打开新标签页，显示详情页 |
| 操作列 | 编辑、删除按钮 |
| 行悬停 | 高亮效果 |

### 1.3 视觉要求

- 列表需要支持玻璃拟态效果
- 列表需要支持主题色

---

## 2. 主表详情页面

### 2.1 URL 结构

详情页在一个新的标签页中打开，URL 应包含记录 ID：
```
/master-detail/orders/:id
```

### 2.2 页面结构

```
┌─────────────────────────────────────────────┐
│ 顶部区域：主表核心信息（只读展示）            │
│ - 订单号、客户名称、订单状态、创建时间等      │
│ - 用卡片包裹                                │
├─────────────────────────────────────────────┤
│ 子表区域                                     │
│ ┌─────────────────────────────────────────┐ │
│ │ 子表1：订单明细（就地编辑表格）           │ │
│ │ - 商品、数量、单价、小计                 │ │
│ └─────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────┐ │
│ │ 子表2：收货地址（弹框编辑）               │ │
│ │ - 收货人、电话、省份、城市、详细地址      │ │
│ └─────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────┐ │
│ │ 子表3：物流信息（抽屉编辑）               │ │
│ │ - 物流公司、运单号、发货状态、预计送达时间 │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

## 3. 子表1：就地编辑表格（Inline Editing）

### 3.1 用途

订单明细：商品、数量、单价、小计

### 3.2 编辑方式

在表格内直接编辑（类似 Excel）

### 3.3 功能详情

| 功能 | 说明 |
|------|------|
| 进入编辑 | 双击单元格进入编辑状态 |
| 新增行 | 底部或顶部有"添加"按钮 |
| 删除行 | 每行有删除按钮 |
| 行内验证 | 数量必须为正整数，单价必须为正数 |
| 实时计算 | 小计 = 数量 × 单价 |
| 撤销修改 | 每行或整体 |
| 已修改标记 | 行背景色变化或侧边标记 |

### 3.4 提交方式

与主表和其他子表一起通过"保存全部"按钮提交

---

## 4. 子表2：弹框编辑（Modal Editing）

### 4.1 用途

收货地址：收货人、电话、省份、城市、详细地址

### 4.2 编辑方式

点击每行的"编辑"按钮，弹出模态框进行编辑

### 4.3 功能详情

| 功能 | 说明 |
|------|------|
| 模态框内容 | 完整表单 |
| 新增地址 | 点击"新增"弹出空表单 |
| 删除地址 | 需二次确认 |
| 设置默认地址 | 单选 |

### 4.4 特点

编辑时聚焦于单条记录，适合字段较多或需要复杂布局的场景

---

## 5. 子表3：抽屉编辑（Drawer Editing）

### 5.1 用途

物流信息：物流公司、运单号、发货状态、预计送达时间、备注

### 5.2 编辑方式

点击每行的"编辑"按钮，从右侧滑出抽屉进行编辑

### 5.3 功能详情

| 功能 | 说明 |
|------|------|
| 抽屉内容 | 表单 |
| 新增、删除 | 支持 |
| 查看物流轨迹 | 可模拟静态数据 |

### 5.4 特点

保留主表上下文的视觉联系，适合需要较大编辑空间但又不希望完全离开页面的场景

---

## 6. 子表公共要求

### 6.1 布局要求

- 每个子表有独立的卡片/区域
- 带标题和操作按钮（新增、刷新等）

### 6.2 功能要求

| 要求 | 说明 |
|------|------|
| 主题色支持 | 子表表格需要支持主题色 |
| 修改标记 | 每个子表显示未保存的修改标记（红点） |
| 行内验证 | 错误时高亮显示并给出提示 |
| 撤销功能 | 支持撤销单行修改或整个子表的修改 |

---

## 7. 保存与验证

### 7.1 保存机制

| 功能 | 说明 |
|------|------|
| 保存全部按钮 | 一次性提交主表和所有子表的修改 |
| 整体验证 | 保存前进行整体验证 |
| 错误定位 | 有错误时定位到对应子表和行 |
| Toast 提示 | 保存成功/失败后显示 |

### 7.2 取消与确认

| 功能 | 说明 |
|------|------|
| 取消编辑 | 支持取消/放弃所有编辑 |
| 离开确认 | 离开页面时如果有未保存修改，弹出确认提示 |

---

## 8. 批量操作

### 8.1 主表列表批量操作

| 操作 | 说明 |
|------|------|
| 批量删除 | 删除选中记录，需二次确认 |
| 批量导出 | 可模拟导出功能，如下载 JSON |

---

## 9. 查询条件区域

### 9.1 详情页顶部（可选）

可放置查询条件，用于筛选子表数据

### 9.2 主表列表页查询条件区域

| 功能 | 说明 |
|------|------|
| 默认展示 | 单行，包含主表关键字段查询、状态筛选、日期范围 |
| 展开更多 | "更多"按钮展开更多条件（至少 3-5 个） |
| 清空/重置 | 支持清空所有条件 |
| 搜索触发 | 查询按钮触发，支持回车快捷键 |

### 9.3 视觉要求

- 查询区域支持玻璃拟态效果

---

## 10. 数据结构

### 10.1 主表（Order）

```typescript
interface Order {
  id: string;
  orderNo: string;
  customerName: string;
  status: 'pending' | 'processing' | 'completed' | 'cancelled';
  createdAt: Date;
  updatedAt: Date;
}
```

### 10.2 子表1：订单明细（OrderItem）

```typescript
interface OrderItem {
  id: string;
  orderId: string;
  productName: string;
  quantity: number;
  unitPrice: number;
  subtotal: number; // 计算字段
}
```

### 10.3 子表2：收货地址（ShippingAddress）

```typescript
interface ShippingAddress {
  id: string;
  orderId: string;
  receiverName: string;
  phone: string;
  province: string;
  city: string;
  address: string;
  isDefault: boolean;
}
```

### 10.4 子表3：物流信息（Logistics）

```typescript
interface Logistics {
  id: string;
  orderId: string;
  company: string;
  trackingNo: string;
  status: string;
  estimatedDelivery: Date;
  remark?: string;
}
```
