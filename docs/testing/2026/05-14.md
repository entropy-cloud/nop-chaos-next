# Manual Test Issues - 2026-05-14

## 1. 登录窗口输入框图标与输入文字重叠

- **所属模块**: `apps/main` / login
- **问题描述**: 登录窗口中的用户名、密码等输入框左侧带图标时，图标与输入文字显示区域重叠，输入后文字压在图标位置上，影响可读性与录入体验。
- **预期行为**: 输入框在有前置图标时应为文字内容预留足够左侧内边距，图标和输入内容不应重叠。
- **发现方式**: 人工诊断
- **状态**: 已修复（`apps/main/src/pages/auth/login/index.tsx` 为带图标输入框显式补齐 `pr-3 pl-10`，避免 `Input` 默认 `px-*` 与局部左侧占位类合并后导致内容压到图标区域）

## 2. mock 模式英文登录后侧边菜单显示为 `menuKey`

- **所属模块**: `apps/main` / mock auth / menu i18n
- **问题描述**: 在 mock 模式下使用英文环境登录进入系统后，侧边菜单项未显示英文文案，而是直接显示 `menuKey` 一类的键名，说明菜单文案解析或国际化回退链路异常。
- **预期行为**: 英文登录进入系统后，菜单应显示对应英文文案；即使缺少翻译，也应优先展示可读标题而不是原始键名。
- **发现方式**: 人工诊断
- **状态**: 已修复（先在 `apps/main/src/main.tsx` 挂载前等待 `initializeI18n()`，随后进一步修正 `packages/core/package.json`，把 `i18next` 声明为 peer，消除 `@nop-chaos/core` 与主应用解析到不同 i18n 单例而导致运行期交互后整站文案退回 key 的问题）

## 3. 进入 Flux Demo 后宿主菜单与登录页退回翻译 key

- **所属模块**: `apps/main` / `@nop-chaos/flux` / host i18n boundary
- **问题描述**: 在英文环境下登录后，如果先进入 `Flux Demo`，等待页面渲染，再切回宿主其他菜单或执行登出，宿主菜单与登录页会重新显示 `menu.dashboard`、`auth.login`、`auth.username` 一类 key。
- **预期行为**: Flux 页面应使用自己的翻译实例而不影响宿主应用；离开 Flux 页面后，宿主菜单、路由页和登录页仍应显示宿主英文文案。
- **发现方式**: 人工诊断
- **状态**: 已修复（宿主根部在 `apps/main/src/main.tsx` 中显式使用 `I18nextProvider` 绑定宿主 i18n，隔离 `@nop-chaos/flux` 在 `initReactI18next` 中对默认实例的覆盖；同时在 `tests/e2e/i18n-persistence.spec.ts` 中补充 `Flux Demo -> 返回宿主 -> 登出` 回归链路）
