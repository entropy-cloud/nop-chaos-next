# Manual Test Issues - 2026-05-19

## 1. master-detail 详情页删除物流后确认弹层关闭后仍拦截点击

- **所属模块**: `apps/main` / `data-management/master-detail/[id]`
- **问题描述**: 在 `http://localhost:4173/#/data-management/master-detail/1001` 删除物流信息时，点击删除后的第一个应用内确认框点 `Confirm` 之后，会再次出现一个标题为 `Confirm`、正文为空的对话框；此时继续点 `Confirm` 或 `Cancel` 都不会产生业务效果。进一步诊断发现，这一表现与取消后看不见的 overlay 挡点击是同一条 confirm 收尾链路上的残留问题。
- **预期行为**: 删除物流时只出现一次有正文的确认框；点 `Confirm` 后应直接删除目标物流记录，不应残留空白 `Confirm` 对话框。
- **发现方式**: 人工诊断
- **状态**: 已修复（`ConfirmDialogHost` 原先即使 `request` 已清空也会继续渲染一个 `open={false}` 的 `AlertDialog`，从而留下标题为 `Confirm`、正文为空的残留对话框；现已改为 `request` 为空时直接 `return null`。同时保留先前对 `AlertDialogOverlay` 关闭态 pointer-events 和表单按钮 `type` 的修正，并由 `tests/e2e/master-detail-dialogs.spec.ts` 锁定"确认后不允许残留空白 Confirm 对话框"。)

## 2. AI Workbench 等页面语义按钮出现深色底配深色文字

- **所属模块**: `packages/theme-tokens` / `--secondary` token 值
- **问题描述**: 在 `http://localhost:4173/#/ai-workbench` 中，"重新生成"按钮（`variant="secondary"`）显示为高饱和紫色背景 + 深蓝色文字（`rgb(166, 137, 250)` + `rgb(10, 71, 169)`），看起来像"深色底配深色文字"。
- **预期行为**: `secondary` 按钮应为浅灰低饱和背景 + 深色文字，shadcn 风格。
- **发现方式**: 人工诊断 + Playwright 诊断 e2e 打印 computed style
- **状态**: 已修复（将所有 4 个主题变体的 `--secondary` 改为浅灰色 `210 40% 96%`/`217 33% 17%`，`--secondary-foreground` 改为与 `--foreground` 一致；诊断 e2e 确认新值为 `rgb(241, 245, 249)` + `rgb(2, 8, 23)`）

## 3. Theme settings 的 Visual theme 选项卡文案未换行导致溢出

- **所属模块**: `apps/main` / `settings/theme`
- **问题描述**: 在 `Theme settings` 页面中，`Visual theme` 下的 `Classic` 和 `Glass` 两个小卡片没有按标题与描述分成两行展示；由于 `Toggle` 默认使用 `inline-flex` 和 `whitespace-nowrap`，描述文本被挤到同一行，出现卡片内文案溢出和截断。
- **预期行为**: 主题选项卡应采用纵向内容流，标题单独一行，描述文本在下一行正常换行，不应溢出卡片边界。
- **发现方式**: 人工诊断
- **状态**: 已修复（在 `apps/main/src/pages/settings/theme/index.tsx` 将主题选项卡改为 `flex-col items-start whitespace-normal`，并为标题/描述补上 `min-w-0`，使文案可以稳定纵向排布和换行；`pnpm --filter @nop-chaos/main exec vitest run src/pages/settings/theme/index.test.tsx` 通过）
