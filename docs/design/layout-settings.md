# 布局设置详细设计

> 本文档描述布局设置模块的交互细节。

---

## 1. 布局设置页面

### 1.1 侧边栏模式设置

| 设置项          | 说明                       |
| --------------- | -------------------------- |
| 侧边栏折叠/展开 | 切换侧边栏的展开和折叠状态 |

### 1.2 侧边栏宽度配置（运行时可调）

用户可以在布局设置页面实时调整侧边栏宽度：

| 配置项   | 范围        | 步进    | 默认值 |
| -------- | ----------- | ------- | ------ |
| 展开宽度 | 14 - 28 rem | 1 rem   | 18 rem |
| 折叠宽度 | 4 - 8 rem   | 0.5 rem | 5 rem  |

**操作按钮**：

- 切换折叠/展开：立即切换侧边栏状态
- 重置宽度：恢复到默认宽度配置

### 1.3 环境变量配置（开发/部署时）

侧边栏宽度支持通过环境变量配置默认值（可选）：

| 环境变量                       | 说明                  | 默认值 |
| ------------------------------ | --------------------- | ------ |
| `VITE_SIDEBAR_WIDTH`           | 侧边栏展开宽度（rem） | 18     |
| `VITE_SIDEBAR_COLLAPSED_WIDTH` | 侧边栏折叠宽度（rem） | 5      |

> 注意：当前项目未在 `.env` 文件中定义这些变量，代码使用默认值。如需自定义，可在 `apps/main/.env` 中添加。

## 2. 侧边栏交互细节

### 2.1 菜单项 Hover 状态区分

侧边栏菜单项根据类型有不同的 hover 效果：

| 菜单项类型           | Hover 效果                   |
| -------------------- | ---------------------------- |
| 分组菜单（有子菜单） | 灰色背景，灰色文字           |
| 叶子菜单（无子菜单） | Primary 色调背景，前景色文字 |

### 2.2 折叠状态下的滚动行为

- 折叠模式下：隐藏右侧 padding，保持紧凑布局
- 展开模式下：保留右侧 padding（pr-1），支持菜单滚动

### 2.3 可访问性合同

- shell 在页面左上角提供 skip link，目标为 `#main-content`
- mobile sidebar overlay 支持点击遮罩关闭，并支持 `Escape` 关闭
- `Sidebar` 使用导航 landmark，并为可展开分组暴露 `aria-expanded`

## 3. 标签页导航交互细节

### 3.1 右键菜单对齐

标签页右键菜单需要智能对齐，避免溢出视口：

| 场景              | 对齐方式                 |
| ----------------- | ------------------------ |
| 右键点击标签页    | 左对齐（align: 'left'）  |
| 点击更多按钮（⋯） | 右对齐（align: 'right'） |

### 3.2 视口边界处理

菜单弹出时自动计算位置，确保：

- 距离视口边缘至少 12px
- 不会超出视口范围

### 3.3 键盘与状态语义

- `TabsBar` 更多操作菜单支持键盘打开与 `Escape` 关闭
- 刷新按钮、更多按钮等 icon-only 控件必须提供可访问名称
- 页面标题通过 `PageHeader` + `useDocumentTitle()` 设置浏览器 document title

---

## 4. 多标签页导航开关

### 4.1 功能说明

提供"是否启用多标签页导航"的开关：

- 启用时：使用顶部多标签页导航
- 关闭时：退化为单页面路由导航

### 4.2 持久化

- 所有布局设置需要被持久化到 localStorage
- 刷新页面后恢复之前的布局状态

### 4.3 Immediate-Apply 合同

- layout settings 页面的侧边栏宽度输入是 immediate-apply 配置，不存在单独保存按钮
- 数字输入清空时不应把宽度写成 `0`；只有可解析数字才会提交到 store
